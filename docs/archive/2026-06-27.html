🧠 Data Science Topic of the Day

🎓 SHAP (SHapley Additive exPlanations)

The mathematical backbone of 'why did the model say that?' — turning black-box predictions into auditable, feature-level explanations.

What It Is

SHAP is a method for explaining the output of any machine learning model by assigning each input feature a 'Shapley value' — a number that represents how much that feature contributed to a specific prediction, relative to a baseline (average prediction). Rooted in cooperative game theory, Shapley values are the only attribution method that simultaneously satisfies three fairness axioms: consistency (features that matter more always get higher values), missingness (absent features get zero credit), and efficiency (all feature contributions sum exactly to the difference between the prediction and the baseline). This makes SHAP the gold standard for explainability — not just a visual aid, but a mathematically provable, auditable accounting of a model's reasoning.

How It Works

1. BASELINE: Establish the expected model output across all training data (e.g., average predicted churn probability = 0.22). 2. COALITION SAMPLING: For each feature, SHAP simulates all possible subsets ('coalitions') of features — with and without the feature in question — measuring how much the prediction changes each time the feature is added. 3. SHAPLEY VALUE: Average the marginal contribution of a feature across all coalitions. This is its Shapley value. 4. SUMMATION: All Shapley values for a single prediction sum to exactly (prediction − baseline). So if the model predicts 0.61 churn for a customer and baseline is 0.22, the Shapley values for all features must sum to +0.39. 5. VISUALISATION: TreeSHAP (optimised for XGBoost/Random Forests, runs in polynomial not exponential time) and KernelSHAP (model-agnostic, slower) are the two dominant implementations. Output formats include: force plots (single prediction waterfall), beeswarm plots (global feature importance across all predictions), and dependence plots (how one feature interacts with another). Key distinction: SHAP produces LOCAL explanations (per prediction) that aggregate into reliable GLOBAL explanations — unlike traditional feature importance which only gives the global view.

When To Use

1. REGULATORY / AUDIT REQUIREMENTS: Any model used in credit decisioning, supplier risk, or KYC must be explainable to regulators. SHAP gives you a documented, per-decision audit trail. 2. MODEL DEBUGGING: When a model underperforms on a segment, SHAP tells you which features are misfiring — far faster than manual ablation testing. 3. SALES INTELLIGENCE: Surface the top 3 reasons a prospect scores highly so a rep understands *why* before the call — not just a score. 4. FEATURE ENGINEERING ITERATION: Low or near-zero SHAP values across all predictions signal features you can safely drop, reducing model complexity. 5. DETECTING DATA LEAKAGE: If a feature like 'customer_id' or a date field shows outsized SHAP values, you have leakage. 6. AVOID when: you need real-time explanations at sub-millisecond latency (SHAP adds compute overhead), or when stakeholders only need a single global ranking and model governance isn't a concern.
