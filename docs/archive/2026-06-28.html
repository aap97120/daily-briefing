🧠 Data Science Topic of the Day

🎓 SHAP (SHapley Additive exPlanations)

Why did the model say that? SHAP gives every prediction a receipt.

What It Is

SHAP is a method from cooperative game theory that assigns each input feature a fair share of the credit (or blame) for a model's output. Rooted in Shapley values — originally developed to split payouts fairly among players in a coalition game — SHAP translates that math into ML: for any single prediction, it tells you exactly how much each feature pushed the score up or down from the baseline average. It works across model types (XGBoost, neural nets, linear models) and produces both local explanations (why THIS prediction?) and global explanations (what does the model rely on overall?).

How It Works

1. BASELINE: Establish the expected model output across the training set — this is the starting point for every explanation. 2. MARGINAL CONTRIBUTIONS: For a given prediction, SHAP iterates over all possible orderings of features and calculates how much each feature changes the output when it is 'added' to the coalition of known features. This is the Shapley value — the average marginal contribution across all orderings. 3. EFFICIENCY GUARANTEE: SHAP values always sum exactly to (prediction − baseline), so the explanation is complete — no unexplained residual. 4. PRACTICAL SHORTCUTS: Brute-force Shapley is exponential, so libraries use smart approximations: TreeSHAP (exact, fast for tree ensembles), LinearSHAP (for linear models), DeepSHAP (for neural nets), and KernelSHAP (model-agnostic, slower). 5. AGGREGATION: For global insight, plot mean absolute SHAP values per feature (importance), or a beeswarm plot showing direction + magnitude across all rows.

When To Use

• Regulated or high-stakes scoring: credit risk, fraud, lead scoring — anywhere a customer or regulator asks 'why?' • Model debugging: SHAP reveals when a model is leaning on a proxy variable (e.g. zip code standing in for industry vertical) • Feature selection: prune low-SHAP features to reduce complexity without hurting accuracy • A/B testing augmentation: when two models score similarly on aggregate metrics, SHAP exposes whether they are reasoning the same way • Monitoring drift: track mean SHAP values over time — if the contribution of 'days since last payment' suddenly spikes, that is a data drift signal before overall accuracy degrades

🎯 Worked Example — D&B Context

D&B trains an XGBoost model to score the probability that a B2B prospect will convert within 90 days — the Prospecting Agent archetype. The model ingests Paydex score, DUNS-verified employee count, corporate family tree depth, years in business, industry sector, and recent trade line activity. Aggregate accuracy looks fine, but a sales rep challenges a low score on a $50M manufacturer that 'feels right.' You run TreeSHAP on that single prediction. The receipt shows: Paydex (+0.18), employee count (+0.12), industry (+0.09), but years in business (−0.22) and recent trade line count (−0.15) are dragging the score down hard. The rep now understands: the company is old and trades conservatively — not a bad sign, just a pattern the model associates with slow decisions. The sales manager uses this to reclassify the account as 'long-cycle, high-value' rather than 'low probability.' At the global level, the same SHAP beeswarm reveals that corporate family tree depth — a proprietary D&B asset — is the third-most influential feature globally, reinforcing the structural moat argument: no competitor can replicate that signal.
